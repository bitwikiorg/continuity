# Usage Guide

## Quick start

1. **Clone** this repo.
2. **Copy** needed templates to your agent workspace:
   ```bash
   cp AGENTS.md IDENTITY.md SELF.md SOUL.md INIT.md STATE.md TODO.md SNAPSHOT.md /your/workspace/
   ```
3. **Fill in** instance-specific values (search for `{{UPPER_CASE}}` placeholders).
4. **Remove** files you don't need (see tiers in `README.md`).
5. **Run** `INIT.md` protocol on first boot.
6. **Delete** `BOOTSTRAP.md` after identity is established.

## Runtime-specific setup

### Prompt-injection frameworks

1. Copy templates to project root or project instructions directory.
2. Keep `AGENTS.promptinclude.md` — it wires files into system prompt.
3. Remove files you don't need from the promptinclude list.
4. Mirror canonical files to project instructions directory for auto-injection.

### Native file-loading frameworks

1. Copy `SOUL.md` to runtime-native home path (auto-loaded every turn).
2. Copy other state files to runtime-native path.
3. Place `AGENTS.md` in project root.
4. Remove `AGENTS.promptinclude.md` (native injection handles loading).

### CLI-based agents

1. Copy `AGENTS.md` and `INIT.md` as project instructions.
2. Import other files as needed.
3. Remove `AGENTS.promptinclude.md`.

### Generic / other

1. Copy needed templates to workspace.
2. Read `AGENTS.md`, then `INIT.md`, then files in load order.
3. Remove `AGENTS.promptinclude.md`.

## Post-boot checklist

- [ ] All `{{PLACEHOLDERS}}` replaced with real values
- [ ] `IDENTITY.md` has name, role, emoji
- [ ] `SELF.md` has operating principles and invariants filled
- [ ] `STATE.md` declares `READY` or `READY_WITH_WARNINGS`
- [ ] `TODO.md` has at least one task or is explicitly empty
- [ ] `SNAPSHOT.md` has timestamp within 7 days
- [ ] Secrets are in `.secrets/` or env vars, NOT in markdown files
- [ ] `BOOTSTRAP.md` deleted (if first-run is complete)

## Customization

### Minimal (essential agents)
AGENTS, IDENTITY, SELF, SOUL, STATE, INIT, TODO, SNAPSHOT

### Standard (most agents)
+ USER, MEMORY, TOOLS, PLAN, CONTEXT, TASKS, LEARNINGS, HEARTBEAT

### Full (operational agents)
+ INFRASTRUCTURE, POTENTIAL, CONTEXT_BUDGET, MISSIONS, BACKLOG, WARNING, BOOTSTRAP, KNOWLEDGE, EVENTS, ARTIFACTS, BOOT

### Advanced (recursive multi-agent)
+ CHECKPOINT, AGENT_ACTIONS, BINDING.schema.json

> **30+ files is heavy. Most agents need 3-5.**
> The full template set exists because different loop types need different state shapes.
> Start with AGENTS.md + TODO.md + STATE.md. Add files only when your agent's loop actually needs them.


## Nested file naming convention

| Directory | Pattern | Example |
|-----------|---------|---------|
| `snapshots/` | `YYYY-MM-DDTHH-MM-SS.md` | `2026-06-27T14-30-00.md` |
| `completed/` | `YYYY-MM-DD-task-slug.md` | `2026-06-27-fix-webhook.md` |
| `journal/` | `YYYY-MM-DD-turn-N.md` | `2026-06-27-turn-3.md` |
| `reflections/` | `YYYY-MM-DD-topic-slug.md` | `2026-06-27-delegation-failure.md` |
| `audits/` | `YYYY-MM-DD-type-slug.md` | `2026-06-27-security-audit.md` |
| `proposals/` | `YYYY-MM-DD-topic-slug.md` | `2026-06-27-api-redesign.md` |
| `setup/` | `YYYY-MM-DD-tool-name.md` | `2026-06-27-caddy-setup.md` |
| `artifacts/` | `YYYY-MM-DD-name.ext` | `2026-06-27-inventory.json` |
| `memory/` | `namespace-topic-slug.md` | `semantic-server-ip.md` |
| `knowledge/` | `topic-slug.md` | `api-rate-limits.md` |

**History dirs:** date-prefixed, append-only, never modify.
**Library dirs:** topic-slug, curate freely.

## Scaffold regeneration

To create the directory structure on a fresh machine:

```bash
./scripts/init-agent.sh /target/directory
```

This creates all subdirectories and `.gitkeep` files. Template files are copied
from the repo, not generated by the script — this keeps templates versioned.
